# Hindi ASR Fine-tuning: Whisper-small

> Segment-based pipeline for fine-tuning OpenAI's Whisper-small on Hindi speech data — covering preprocessing, training, evaluation, and error analysis.

---

## Overview

This project fine-tunes [`openai/whisper-small`](https://huggingface.co/openai/whisper-small) on a custom Hindi speech dataset (JoshTalks) using a **segment-level pipeline**. Instead of feeding full audio recordings (7–20 minutes each) to the model, we extract timestamped segments (~9,500 examples from 104 recordings) to align with Whisper's 30-second input constraint.

| Detail | Value |
|---|---|
| **Model** | openai/whisper-small |
| **Language** | Hindi (Devanagari) |
| **Task** | Automatic Speech Recognition (ASR) |
| **GPU** | NVIDIA GTX 1650 (4GB VRAM) |
| **Evaluation Set** | FLEURS Hindi (`hi_in`) |

---

## Results

| Model | Training Data | WER (%) |
|---|---|---|
| Whisper-small (Pretrained Baseline) | None — zero-shot | 67.52% |
| Whisper-small (Fine-tuned) | JoshTalks Hindi (~9,500 segments) | **Improved** |
| Whisper-small + Number Normalization | JoshTalks Hindi + post-processing | **Best** |

> WER = Word Error Rate. Lower is better.

## Key Design Decisions

### Segment-level training (not full audio)
Whisper processes a maximum of 30 seconds of audio at a time. Since the raw recordings are 7–20 minutes long, we extract individual timestamped segments from the transcription JSONs. This converts 104 recordings into ~9,500 training examples and provides properly aligned audio-text pairs.

### Train/validation split by recording ID
The split is done at the **recording level**, not the segment level. This prevents data leakage where segments from the same speaker/recording appear in both train and validation sets.

### Encoder frozen during fine-tuning
To fit within 4GB VRAM, the Whisper encoder is frozen and only the decoder is fine-tuned. This significantly reduces memory usage and training time while still adapting the model to Hindi.

### Hindi text normalization
All text is NFC-normalized, stripped of non-Devanagari characters (punctuation, English, symbols), and whitespace-collapsed before training and evaluation.

## Error Analysis

Errors are sampled systematically using **stratified sampling by WER severity**:

- **Low** (WER < 0.3) — 9 samples
- **Medium** (0.3 ≤ WER < 0.7) — 9 samples
- **High** (WER ≥ 0.7) — 9 samples

**Top error categories identified:**
1. Phonetic substitution in complex Hindi words
2. Function word deletion (है, का, में, etc.)
3. Numeral misrecognition (digits not converted to words)

---

## Proposed Fixes

| Fix | Error Type | Approach |
|---|---|---|
| Fix 1 | Function word deletion | Hindi n-gram LM shallow fusion (KenLM) |
| Fix 2 | Phonetic substitution | Data augmentation + speed perturbation |
| Fix 3 | Numeral misrecognition | Post-processing with `num2words` (Hindi) |

Fix 3 is **implemented** in Part G of the notebook with before/after WER comparison.
